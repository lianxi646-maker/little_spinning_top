底盘：

前提说明：底盘用的单环速控；

整体思路：初始化PID——>对云台yaw轴反馈角度进行前馈处理——>计算底盘跟随所需的补偿角速度——>将补偿角速度添加到算出的底盘整体角速度——>将底盘整体速度解算至各轮速度——>PID计算；

云台yaw轴的反馈相对角度直接沿用框架中的DM电机结构体中的相对角度；

底盘跟随补偿角速度是使将反馈相对角度的编码器值放入位置环PID计算得来；

底盘原本角速度是通过yaw轴反馈角计算相对旋转矩阵乘上遥控器映射的整体速度得来；

解算则直接使用的框架中的麦轮解算；

由于遥控器拨杆不够用，故有两种控制方式，一是小陀螺与底盘跟随均具有但pitch电机不可动，二是只有底盘跟随但无小陀螺，pitch电机可动，并且加入离线保护（当状态在以上两种之外时保留上一次的姿态）。

云台：

前提说明：云台用到双环控制。该云台算法用到了麻神所讲的用陀螺仪解算高精度云台相对于底盘转速（实际上就是电机转速）。该算法为实验算法，若后续上车调试效果不好则会直接引用电机反馈转速

整体思路：初始化PID——>更新云台相对于底盘角度——>解算云台电机转速——>解算云台补偿角速度并结合遥控器数值映射云台控制速度——>PID计算；

pitch轴云台相对于底盘的角度只是为了解算云台转速准备的，除此之外无任何作用；yaw轴云台相对于底盘角度直接引用的框架中DM结构体的相对角度；两相对角度都做了归化处理（归化到[-Π，Π]）；

解算云台转速用到的主要思路是用"底盘相对于各云台电机的角速度加上云台自身角速度（所求解的未知变量）等于出云台的绝对角速度（相对于世界坐标系的角速度）"这个等式列出yaw与pitch的等式联立求解；

云台自稳是根据底盘角速度与底盘陀螺仪反馈的x、y轴角速度来计算的，yaw轴自稳主要服务于小陀螺，pitch轴自稳主要服务于上坡时的倾角；

遥控器的控制方法与底盘相同，不再赘述。

板间通信：

主要代码位于新增文件Board2board.c中

该板间通信用CAN通信分四帧解算，帧头分别为0x225、0x226、0x227、0x228。由于陀螺仪解算云台转速所需数据内存量过大所以帧数很多。发送运用八个一字节数组进行通过位运算储存，其中用到了memcpy()函数确保float类型数据能在二进制编码不变的情况下存进数组，接收用共用体内存共用原理来确保各数据接收时二进制编码不变，一定程度上保证了收发相同。

离线检测：

只写了板间通信与遥控器的离线检测。由于框架中的离线检测包含发射电机与超电的离线检测，可能导致程序判断整车一直离线，故板间通信与遥控器的离线检测为我自己添加；

为尽量不改动框架，我将遥控器的离线检测放在了底盘任务源文件中，板间通信离线检测则正常放在板间通信源文件中。

检测发生异常后执行的失能任务位于error_task.c。
